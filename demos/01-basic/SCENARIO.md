# Demo 01 — Basic intent to Frida agent

## What this shows

HOOKCRAFT reads a small YAML **intent** (`intent.yaml`) that declares *what*
you want to instrument in a target process, and generates a ready-to-run
**Frida JavaScript agent**. No Frida or network access is required to generate
the script — only Python 3.10+ (standard library only).

The demo intent targets a hypothetical Android banking app and declares four
hooks of three different kinds:

| name                   | kind            | what it does                                  |
|------------------------|-----------------|-----------------------------------------------|
| `build_auth_token`     | `java_method`   | hooks a Java method (with a specific overload), logs args + return + a Java stack trace |
| `ssl_verify`           | `native_export` | attaches to `SSL_CTX_set_verify` in `libssl.so` |
| `strstr_probe`         | (inferred)      | `kind` omitted; inferred as `native_export` from the `symbol` field |
| `native_plugin_loaded` | `module_init`   | waits for `libpayment.so` to load, reports its base address |

## Run it

Generate and print the agent:

```
python -m hookcraft generate demos/01-basic/intent.yaml
```

Write it to a file:

```
python -m hookcraft generate demos/01-basic/intent.yaml -o hooks.js
```

Lint for CI (machine readable):

```
python -m hookcraft lint demos/01-basic/intent.yaml --format json
```

Then, on a device with Frida set up (outside this demo):

```
frida -U -f com.example.bank -l hooks.js --no-pause
```

## Expected result

- `generate` prints valid Frida JS beginning with the banner
  `Generated by HOOKCRAFT v1.0.0` and containing one block per hook
  (`Interceptor.attach`, `Java.perform`, a module-load observer, etc.).
- The generated agent calls `installHooks()` and ends with a
  `send({ _hookcraft: 'installed', count: 4 })` line.
- `lint` reports findings. This intent is **clean of errors**, but it surfaces
  one `info` finding: `strstr_probe` has no `module`, so the symbol is resolved
  across all modules.
- Exit code is `0` (no error-level findings). If you edit the intent to remove
  a required field (e.g. delete `symbol:` from `ssl_verify`), `lint`/`generate`
  exit non-zero — suitable for a CI gate.
